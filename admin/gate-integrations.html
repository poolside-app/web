📋 Review checklist when a request lands

  1. Triage — is this protocol something we can speak? (TCP/IP HTTP yes, OSDP feasible, Bluetooth-only RFID probably no.)
  2. Risk assessment — vendor's documentation? closed-source binary? prior art on GitHub? Estimated dev time.
  3. Quote — pick: (a) Decline · (b) Accept at standard $250+$25 if you're confident · (c) Custom quote $500–$2,000+ for new development
  4. Pilot — when accepted, schedule on-site visit. Test on tenant's actual panel. Get sign-off.
  5. Promote — once working, promote the request to a Verified Template so future clubs with the same panel are one-click.

📋 Provisioning workflow

  1. Requested → tenant clicked "Request integration." Stripe Checkout link sent automatically.
  2. Paid → setup fee landed. Bridge appears in your queue here.
  3. Provisioning → flash Pi with bridge code, embed tenant_id + auth token, sticker-label, box.
  4. Shipped → click Mark Shipped, paste tracking number. Tenant gets email with setup guide.
  5. Active → bridge auto-detects tenant on first connect, status flips to active. Members can unlock.
